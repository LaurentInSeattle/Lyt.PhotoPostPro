Spikes in your color histogram after gamma correction usually indicate pixel bunching (multiple original tonal values are mapped to the same new value) or bit-depth limitations. This process compresses or expands tones non-linearly, leading to gaps and sudden jumps in the histogram that can result in visible banding or posterization.Why Gamma Spikes HappenTone Compression/Expansion: Gamma correction uses a power-law function: \(V_{out} = A \cdot V_{in}^{\gamma}\). If γ < 1, the function expands dark tones and compresses bright tones, causing low-end pixels to bunch together. If γ > 1, it does the opposite.Binning Effects: Because you are mathematically transforming floating-point values into limited discrete steps (like 256 levels in 8-bit color), consecutive pixel values may be rounded into the same integer, creating tall, vertical spikes.Bit Depth Loss: If you are working in an 8-bit color space (0-255), there simply aren't enough distinct tonal values to stretch the image without leaving gaps.

How to Fix & Prevent the ArtifactsIncrease Bit Depth: Always edit your images in 16-bit color instead of 8-bit. This provides 65,536 tones per channel, leaving plenty of data to perform non-linear adjustments without gaps or spikes.Convert Early, Process in 32-bit: If working with digital composites or video, ensure your working space is set to a linear format, or use a 32-bit float pipeline prior to baking in color corrections.Use Adjustment Layers: In tools like Adobe Photoshop, use Adjustment Layers rather than applying filters directly to the base image. This keeps calculations dynamic and allows the software to pull from the original, uncompressed colors.Dithering: If you must output to 8-bit or lower, introduce a small amount of dither (random noise). This breaks up the solid blocks of color and tricks the eye into seeing a smooth gradient, effectively hiding the gaps in the histogram.

In SixLabors.ImageSharp, white balance is achieved by iterating through the image's pixels and applying a scaling 
multiplier to the Red, Green, and Blue (RGB) channels. 
This corrects color casts by forcing a targeted neutral or white reference area in the image to have 
balanced R, G, and B values.The easiest way to perform a custom white balance is by sampling a known neutral pixel 
in your image and calculating your multipliers using the formulas below.The Mathematics of White BalanceWhite 
balancing scales the color channels relative to a chosen target white or neutral gray. 
If a neutral pixel has an original RGB value of \(R_o, G_o, B_o\), and you want to balance 
it so that it is perfectly neutral (where \(R_{wb} = G_{wb} = B_{wb}\)), the multipliers are defined as follows:
Let the target luminance \(Y = \text{max}(R_o, G_o, B_o)\).
The scaling coefficients are:\(A_1 = \frac{Y}{R_o}\)\(A_2 = \frac{Y}{G_o}\)\(A_3 = \frac{Y}{B_o}\)
Then, for every pixel in the image, the new values are:\(R_{wb} = A_1 \times R_o\)\(G_{wb} = A_2 \times G_o\)\(B_{wb} = A_3 \times B_o\)(Note: These are usually bounded to a maximum of 255 for 8-bit images).Implementation ExampleUse ImageSharp's high-performance ProcessPixelRows API to directly mutate pixel data in C#:csharpusing SixLabors.ImageSharp;

using SixLabors.ImageSharp.PixelFormats;
using SixLabors.ImageSharp.Processing;

public void BalanceWhite(string imagePath, Rgb24 referenceColor, Rgb24 targetNeutral)
{
    // Load the image
    using (Image<Rgb24> image = Image.Load<Rgb24>(imagePath))
    {
        // Calculate multipliers: e.g., target / reference
        // (Make sure to handle division by zero in production code)
        float rScale = (float)targetNeutral.R / referenceColor.R;
        float gScale = (float)targetNeutral.G / referenceColor.G;
        float bScale = (float)targetNeutral.B / referenceColor.B;

        // Process pixels
        image.ProcessPixelRows(accessor =>
        {
            for (int y = 0; y < accessor.Height; y++)
            {
                Span<Rgb24> pixelRow = accessor.GetRowSpan(y);
                for (int x = 0; x < pixelRow.Length; x++)
                {
                    ref Rgb24 pixel = ref pixelRow[x];

                    // Scale the channels, ensuring we don't exceed 255
                    int newR = (int)(pixel.R * rScale);
                    int newG = (int)(pixel.G * gScale);
                    int newB = (int)(pixel.B * bScale);

                    pixel.R = (byte)Math.Min(newR, 255);
                    pixel.G = (byte)Math.Min(newG, 255);
                    pixel.B = (byte)Math.Min(newB, 255);
                }
            }
        });

        // Save the result
        image.Save("balanced_image.jpg");
    }
}

To treat high-ISO grain (luminance and chroma noise) effectively, a pure median blur 
can sometimes look too blocky. A better approach in ImageSharp is to combine a mild 
Gaussian Blur (to blend the grain) with a subtle Box Blur or a very low-radius 
Median Blur, or to apply the processing specifically to a cloned layer to blend it back.

Here is an optimized approach for high-ISO grain using a gentle Gaussian smoothing:C# Implementationcsharpusing SixLabors.ImageSharp;
using SixLabors.ImageSharp.Processing;

// Load the high-ISO photo
using var image = Image.Load<Rgba32>("high_iso_input.jpg");

image.Mutate(x => {
    // Apply a soft Gaussian blur to smooth out the grain
    // Sigma controls the amount of smoothing (keep it low: 0.5 to 1.5)
    x.GaussianBlur(sigma: 0.8f);
});

// Save the cleaner image
image.Save("high_iso_denoised.jpg");


// Load your input image
using var image = Image.Load<Rgba32>("input.jpg");

// Apply the Median Blur
// A radius of 3 is a good starting point for mild to moderate noise
image.Mutate(x => x.MedianBlur(radius: 3));

// Save the processed image
image.Save("output_denoised.jpg");

//Adjusting the Radius: 
// The radius determines the size of the neighborhood window used for filtering. 
// Use a smaller odd number (e.g., 3 or 5). 
// Using too large of a radius can result in a loss of fine details and a "watercolor" 
// effect

Tips for High-ISO GrainKeep Sigma Low: A sigma value between 0.5f and 1.2f targets 
fine high-ISO grain without destroying shapes.
Chroma Noise: ImageSharp operates on RGB space by default. 
If your high-ISO noise has severe colored specks (chroma noise), a small MedianBlur(1) 
or MedianBlur(2) works better than Gaussian blur to eliminate the color artifacts.

    1. The Photographic Exposure EquationExposure is the mathematical product of three camera variables
    (the Exposure Triangle):\(\text{Exposure} \propto \text{Aperture Area} \times \text{Shutter Speed} \times \text{ISO Sensitivity}\)
    To change the exposure by a specific number of stops, you adjust any of these three values:
    Aperture (f-stop): Controls the size of the lens opening. The area is proportional to 
    \(\frac{1}{N^{2}}\) (where \(N\) is the f-number). 
    To let in twice as much light (or add \(+1 \text{ stop}\)), you multiply the aperture denominator 
    by \(\frac{1}{\sqrt{2}}\) (approx \(0.7\)). 
    Common full stops include:\(f/1.4 \rightarrow f/2 \rightarrow f/2.8 \rightarrow f/4 \rightarrow f/5.6 \rightarrow f/8 \rightarrow f/11 \rightarrow f/16 \rightarrow f/22\)
    Shutter Speed (\(t\)): Controls the duration of light exposure. 
    Measured in fractions of a second, doubling the time (e.g., from \(\frac{1}{60}\) to \(\frac{1}{30}\)) adds \(+1 \text{ stop}\) of light.ISO (\(S\)): Controls sensor sensitivity. Doubling the ISO (e.g., \(100 \rightarrow 200\)) adds \(+1 \text{ stop}\) of exposure.2. Math for Post-Processing (Digital Editing)If you are adjusting a photo's exposure in editing software like Photoshop or Lightroom, the math scales linearly with pixel values.For an 8-bit image (where pixel values range from \(0\) to \(255\)), you apply the following formula:\(\text{New Pixel Value} = \text{Original Pixel Value} \times 2^{\Delta EV}\)Where \(\Delta EV\) is the change in Exposure Value (stops).Example: To increase the exposure by \(+1 \text{ stop}\) (\(\Delta EV = 1\)), multiply the original pixel brightness by \(2^{1}\) (\(2\)).Example: To decrease exposure by \(-1 \text{ stop}\) (\(\Delta EV = -1\)), multiply by \(2^{-1}\) (\(0.5\)).3. Exposure Value (EV) FormulaIf you want to calculate the exact mathematical Exposure Value (EV) of a scene, 
    you use a logarithmic equation relating your settings to scene brightness:\(EV = \log_2 \left( \frac{N^2}{t} \right)\)
    
    Where:\(N\) is the aperture f-number\(t\) is the shutter speed in seconds
